## Project Overview

meshmap.net is a real-time map of Meshtastic LoRa mesh network nodes observed on the official Meshtastic MQTT server. It consists of:

- **`meshobserv`** — a Go binary that subscribes to MQTT, decrypts and parses Meshtastic Protocol Buffer messages, maintains an in-memory node database, and periodically writes it as JSON
- **Website** — a vanilla JS + Leaflet single-page app served by Nginx; no build step required
- **Docker Compose** — two services: `meshobserv` and `nginx`

## Build & Run Commands

### Generate Protocol Buffers (required before Go build)
```bash
bash scripts/generate-pb.sh
```
This clones `meshtastic/protobufs` from GitHub via Docker and compiles `.proto` files into `internal/meshtastic/generated/`.

### Build meshobserv Docker image
```bash
bash scripts/build-meshobserv.sh
```
Calls `generate-pb.sh` first, then does a multi-stage Docker build.

### Run with Docker Compose
```bash
cp .env.sample .env   # then edit as needed
docker-compose up -d
# Access at http://localhost:${HTTP_PORT:-10081}
```

### Go build/lint (without Docker, requires protobufs already generated)
```bash
go build ./...
go vet ./...
```

There are no tests in this project.

## Architecture

### Data Flow
1. `meshobserv` connects to the MQTT broker and subscribes to 8 topic patterns covering all Meshtastic regions/channels
2. Incoming `ServiceEnvelope` messages are validated by regex, decrypted with the Meshtastic default AES key, and parsed as Protocol Buffers
3. Per-node data is stored in `NodeDB` (a `map[uint32]*Node`) protected by a `sync.Mutex`
4. Every minute: stale nodes/neighbors/metrics are pruned, and valid nodes are serialized to `website/nodes.json`
5. Nginx serves `website/` (including `nodes.json`) as static files with 60-second cache
6. The browser fetches `/nodes.json` every 65 seconds and re-renders the Leaflet map

### Go Package Structure
- `cmd/meshobserv/meshobserv.go` — main entry point; message dispatch (`handleMessage`), rate limiting, blocklist, MQTT client setup, prune/write loop
- `internal/meshtastic/mqtt.go` — MQTT connection, topic subscription, message decryption, envelope parsing
- `internal/meshtastic/node.go` — `Node` struct with update methods, `NodeDB` type with `Prune`/`WriteFile`/`LoadFile`/`GetValid`
- `internal/meshtastic/generated/` — generated protobuf Go code (gitignored; regenerated by `generate-pb.sh`)

### Key Constants (`cmd/meshobserv/meshobserv.go`)
| Constant | Value | Meaning |
|---|---|---|
| `NodeExpiration` | 21600s (6hr) | Remove node position after 6hr without update |
| `NeighborExpiration` | 7200s (2hr) | Remove LoRa neighbor links after 2hr |
| `MetricsExpiration` | 7200s (2hr) | Remove telemetry after 2hr |
| `RateLimitCount` | 300/min | Per-node message rate limit |
| `PruneWriteInterval` | 1 min | How often NodeDB is pruned and written to disk |

### Message Types Handled
`POSITION_APP`, `NODEINFO_APP`, `TELEMETRY_APP` (device + environment), `NEIGHBORINFO_APP`, `MAP_REPORT_APP`, `TEXT_MESSAGE_APP` (logged only)

### Frontend (`website/index.html`)
Single HTML file with inline CSS and JS (~467 lines). No framework, no build step. Key libraries loaded from CDN:
- Leaflet (map rendering)
- Leaflet.markercluster (node clustering)

State persisted via `localStorage` (theme, map center/zoom) and URL hash (selected node, e.g. `#!deadbeef`).

## Configuration

Runtime config via environment variables (see `.env.sample`):
- `MQTT_BROKER`, `MQTT_USERNAME`, `MQTT_PASSWORD` — MQTT connection
- `HTTP_PORT` — Nginx listen port (default `10081`)
- `SITE_TITLE`, `SITE_URL` — injected into `index.html` via `envsubst` at container startup

The blocklist file (`configs/blocklist.txt`) contains one node ID (uint32, decimal) per line. Pass its path with `-b` flag.

## Important Implementation Details

- **Protobuf generation**: The `internal/meshtastic/generated/` directory is gitignored and must be regenerated before building Go code. `build-meshobserv.sh` handles this automatically.
- **Encryption**: Messages are decrypted using `meshtastic.DefaultKey` (the Meshtastic public default AES key) via `meshtastic.NewBlockCipher`.
- **SeenBy cap**: Each node tracks at most 10 most-recent MQTT topics that reported it.
- **Neighbor cap**: Each node tracks at most 100 most-recent LoRa neighbors (with SNR).
- **Watchdog**: If no MQTT messages are received during a prune interval, `meshobserv` calls `log.Fatal` and exits (triggering Docker restart).
